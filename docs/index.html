tripwire throws a catalog of real agent attacks — prompt injection, data exfiltration, unauthorized tool calls — at a test agent that is built to fall for all of them. Between that agent and its dangerous tools sits a separate gate whose first decision is always no: an action runs only when an explicit rule allows it, and anything unclear is refused and handed off. Across 108 attacks the gate allowed zero unauthorized actions. Remove the gate and the same run lets through every one.

§ 01

Read the
100% honestly

A gate whose first line is verdict = DENY, tested against a corpus of attacks that all break the policy, should block 100% of them. That number is the expected, correct behavior of deny-by-default — not a leaderboard win. The value is elsewhere: the block at the action layer is a by-construction guarantee checked by property tests, every decision lands in a tamper-evident audit log, and an ungated baseline proves the attacks are real by letting every one through. tripwire measures two different defenses and reports them separately — never blended into one score.

§ 02

The gate,
literally

The agent only proposes a tool call; it can never reach a dangerous tool itself. A separate verifier decides, and the only way to execute a tool is a single-use token the verifier mints from an allow decision. No token, no call. Everything else is refused and escalated to a human.

An independent verdict

“Independent” means independent of the agent's story: a separately-typed request, gate-fetched trusted facts, no trust in agent-supplied text. Any internal error, timeout, or malformed policy maps to DENY.

A second, weaker line

After a reply is composed, an egress filter scans it for leaked PII / secret / config shapes and withholds it if any appear. This is defense-in-depth, not the guarantee — see § 03.

§ 03

Two defenses,
reported apart

The 108 defenses are not one number. They are two very different guarantees, and tripwire keeps them separate on purpose: one is proven, the other is best-effort.

Surface 1 · the proof

87 / 87 action authorization — blocked

No high-stakes action runs without an explicit matching rule. This is the claim tripwire actually proves: it holds by construction, is checked by Hypothesis property tests (INV-1…INV-9), and every decision is written to the audit log.

Surface 2 · best-effort

21 / 21 output disclosure — caught

A detection-based egress filter withholds responses that carry PII / secret / config shapes. It is a shape filter — bypassable by a determined encoder. tripwire puts the guarantee at the action layer, where it can be proven, and treats output filtering as a weaker second line.

§ 06

What the numbers
do and don't claim

The guarantee is about what the gate lets execute, not the wording of the agent's replies. Swap the obedient mock for a real model and it would attempt fewer attacks; the allowed-unsafe number would still be enforced by the gate, not the agent.

action authorization: 87 / 87  (the real claim)

No high-stakes action runs without an explicit matching rule. True by construction, checked by property tests INV-1…INV-9, and recorded in the audit log.

100% defense  (expected, not a score)

Deny-by-default against a corpus that all violates the policy is supposed to reach 100%. It's the correct behavior of the design, not a benchmark leaderboard result.

the agent is worst-case obedient  (on purpose)

The mock obeys every instruction it reads, in any channel. That's deliberate: the harness measures what the gate prevents, not how clever the agent is. A cautious real agent can only do better.

output filter: 21 / 21  (best-effort)

The disclosure catches come from a shape-matching egress filter — bypassable by a determined encoder. The guarantee lives at the action layer, on purpose; this is a weaker second line.
